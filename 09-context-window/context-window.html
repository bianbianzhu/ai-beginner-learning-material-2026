📏 Context Window 可视化

每次调 responses.create，OpenAI 都会把 全部 input + 全部 output（包括 tool call、tool output、reasoning） 放进一个有限大小的"口袋"里——这个口袋就叫 context window。 口袋装不下就会报错或被截断。

下面用总窗口 128,000 tokens（gpt-5.4-nano 的大致容量）做例子。每个彩色块代表一种消息类型占的 token 数。

⚠️
💡 三条必须记住的规律：
  1. 工具 output 是最大的消耗者（黄色块总是最长）。尤其是返回 JSON 数据的工具。
  2. 轮次越多 → 空闲越少。无脑把 history 一直往数组里 push，迟早爆炸。
  3. 当窗口快满时有 3 种救场法：
    • 🪚 截断（trim）：删掉最老的几轮 — 简单但可能丢关键信息
    • 🗜️ 摘要（summarize）：用模型把老历史压成一段摘要
    • 🏗️ 服务端托管：用 2026 的 Conversations API 或 previous_response_id + context_management / 自动 compaction，让 OpenAI 帮你管

🧮 自己估算：token 粗略换算

  • 英文：1 个 token ≈ 0.75 个单词（100 单词 ≈ 133 token）
  • 中文：1 个汉字 ≈ 1~2 个 token
  • JSON：字段名 + 引号 + 花括号都吃 token —— 结构越复杂越烧
  • 精确算：去 platform.openai.com/tokenizer
