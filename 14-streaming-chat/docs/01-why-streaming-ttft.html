⏱️ 为什么要 Streaming? —— TTFT 的意义

后端明明已经能调用模型了，为什么 ChatGPT 那种"边打字边显示"的体验还需要专门学？
答案只有一个词：TTFT（Time to First Token）。

🐢 Non-streaming：用户要等全部生成完才看到任何东西

用户视角：点下"发送"到看到文字的时间 总耗时 3.2s
→
⏳ 沉默……AI 在后台生成（用户不知道发生了什么）
💥 一次性出全部
0s1s2s3s

🚀 Streaming：首字在 200ms 内出现，之后持续流出

用户视角：200ms 就看到第一个字开始打出来 总耗时 3.2s（一样）
→
首字
持续 token by token 流出 ——用户一边看一边读
0s1s2s3s
TTFT ≈ 200ms Time to First Token —— 用户感知延迟的唯一关键指标

🧠 关键认知：响应速度 ≠ 总时长

① 感知延迟由 TTFT 决定

用户"什么时候看到第一个字"，比"总共花多久"重要得多。
200ms 首字的 3 秒回复，比 2 秒沉默的 2 秒回复体验更好。

② 用户边看边读

正常人阅读速度 ≈ 200 wpm（每秒 3-4 个词）。
如果 AI 生成速度 ≥ 阅读速度，用户根本不会"等"——他们在读。

③ 系统"活着"的信号

流式 = 每一个 chunk 都在告诉用户"我还在工作"。
没有 loading spinner 也不会被误以为卡死。

📊 关键指标一览

TTFT = First-token latency
TPS = Tokens / second（生成速率）
Total latency = TTFT + Output / TPS

典型值（gpt-5.4-nano 同区域）：TTFT ≈ 150-400ms，TPS ≈ 80-150 tok/s。跨区域加 RTT。

💡 一句话记住本课的动机：

非流式：用户等 3 秒，什么都看不到 → 像卡死的页面。
流式：首字 200ms 出现 → 用户立刻获得"系统在工作"的反馈 + 可以提前阅读。

所以 AI 产品的体验差距，很大一部分就在要不要做 streaming。
