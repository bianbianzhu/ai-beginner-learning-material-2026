🌊 Responses API 流式事件全景

调用 client.responses.create({ ..., stream: true }) 得到的不是一个对象，是一个语义事件流。
每个事件都有 type 字段。你用 for await (const ev of stream) 遍历，按 type 做分发。

生命周期事件
文本流
工具调用开始
工具参数流
工具调用完成
整体完成

📜 分支 A：只生成文本（没有 tool call）

response.created 1次
{ response: { id, model, status:"in_progress" } }
response.output_item.added
{ item: { type:"message", role:"assistant", content:[] } }
response.content_part.added
{ part: { type:"output_text", text:"" } }
response.output_text.delta N 次 · 核心！
{ delta: "你" } { delta: "好" } { delta: "，" } { delta: "我" } ...
response.content_part.done
{ part: { type:"output_text", text:"你好，我是 ..." } }
response.output_item.done
item 最终态（完整 message）
response.completed 1次
{ response: { status:"completed", usage:{ input_tokens, output_tokens } } }

🔧 分支 B：模型决定调用 tool（例："东京几度？" → get_weather）

response.created
{ status:"in_progress" }
response.output_item.added item.type === "function_call"
{ item: { type:"function_call", name:"get_weather", call_id:"call_abc", arguments:"" } }
response.function_call_arguments.delta N 次 · 拼 JSON 字符串
{ delta: "{\"ci" } { delta: "ty\":" } { delta: "\"Tok" } { delta: "yo\"}" }
response.function_call_arguments.done
{ arguments: "{\"city\":\"Tokyo\"}" } ← 完整 JSON 字符串拿到
response.output_item.done
item 终态，你现在可以 JSON.parse(arguments) 然后本地执行 get_weather
response.completed
注意：这一轮 response 到此结束。你要再发一次 responses.create 带着 function_call_output 继续。

🔁 分支 C：完整 agent 循环（Lesson 12 流式版）

while (true) {
  const stream = await client.responses.create({
    model: "gpt-5.4-nano",
    input: history,
    tools,
    stream: true,
  });

  const pending = new Map();   // call_id → { name, args:"" }
  let text = "";
  let hadToolCall = false;

  for await (const ev of stream) {
    switch (ev.type) {
      case "response.output_text.delta":
        text += ev.delta;
        writeSSE("delta", { text: ev.delta });
        break;

      case "response.output_item.added":
        if (ev.item.type === "function_call") {
          hadToolCall = true;
          pending.set(ev.item.id, { name: ev.item.name, args: "", call_id: ev.item.call_id });
          writeSSE("tool_start", { id: ev.item.id, name: ev.item.name });
        }
        break;

      case "response.function_call_arguments.delta":
        pending.get(ev.item_id).args += ev.delta;
        writeSSE("tool_args_delta", { id: ev.item_id, delta: ev.delta });
        break;

      case "response.output_item.done":
        if (ev.item.type === "function_call") {
          const parsed = JSON.parse(ev.item.arguments);
          const result = await runTool(ev.item.name, parsed);
          history.push(ev.item);
          history.push({ type:"function_call_output", call_id: ev.item.call_id, output: JSON.stringify(result) });
          writeSSE("tool_result", { id: ev.item.id, result });
        }
        break;
    }
  }

  if (!hadToolCall) {
    history.push({ role:"assistant", content: text });
    writeSSE("done", {});
    return;          // ← 没有更多 tool，退出循环
  }
  // ← 有 tool call，回到 while 顶部重发一次 responses.create
}
💡 你只需记住这 6 个事件（按出现顺序）：
  • response.created — 响应开始（可做"开始"UI 提示）
  • response.output_text.delta — 每个文字碎片 · N 次 · 追加到 assistant 消息
  • response.output_item.added — 新 item（可能是 message，也可能是 function_call）
  • response.function_call_arguments.delta — 每个 JSON 字符碎片 · 拼到 args buffer
  • response.output_item.done — 这个 item 写完了（function_call 要在这里执行）
  • response.completed — 整个响应结束（带 usage；可能仍需下一轮循环）

⚠️ 文本流和工具流可以穿插出现——同一个 response 里，AI 可能先说两句话，再调一个工具。switch 各种 type，别假设顺序。
