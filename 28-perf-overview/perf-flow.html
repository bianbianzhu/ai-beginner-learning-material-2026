OpenAI 官方的 7 个 Latency 原则

1 Process tokens faster

用更小的模型 / 蒸馏 / 推理优化（如 Predicted Outputs）。本课暂不做 demo。

2 Generate fewer tokens

输出越短越快。让模型「简洁回答」、缩短 JSON key。

3 Use fewer input tokens

主要靠 Prompt Caching（静态前缀放前面）。

→ Lesson 30
4 Make fewer requests

能合并的多步合成一个 prompt，省掉多次 round-trip。

→ Lesson 31
5 Parallelize

独立步骤用 Promise.all 同时发，总耗时 = 最慢那个。

→ Lesson 31
6 Make your users wait less

Streaming 把感知延迟从 5s 缩到 0.6s。

→ Lesson 29 + 32
7 Don't default to an LLM

能写死的写死、能预算的预算、能 hash 表搞定的别叫 LLM。

📌 三件你要记住的事

1. AI 性能问题，本质是系统问题。

2. 优化顺序：减少计算 → 改善体验 → 控制成本。

3. 好的 AI 产品，让用户「等得住」。没有指标的优化，基本都是浪费时间。
